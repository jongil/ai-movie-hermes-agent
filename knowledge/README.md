# knowledge — 시사베테랑 지침·지식 허브

시사베테랑 채널의 **정본 지침(작성 시스템)·지식(제작 가이드)** 을 버전 고정해 모아둔 곳.
작가(writer)뿐 아니라 **디렉터**도 검수·위임에 공용으로 참조한다.

이중 역할:
1. **정본 지침/지식** (`*.md`, 버전 고정 스냅샷) — 채널 운영 규칙의 단일 출처. 프로필 안의 운영 사본
   (`profiles/writer/REFERENCE_writing_system.md` 등)은 이 정본의 배포 미러.
2. **증류 원본** (`sources/`, git 제외) — ebook·PDF에서 실행가능 원칙을 뽑아 지침에 반영하기 위한 재료.

## sources/ — 증류 원본 (git 제외)

| 경로 | 내용 | git |
|------|------|-----|
| `sources/` | ebook·PDF 등 원본 레퍼런스 (바이너리·대용량) | **제외** (`.gitignore`) |

### 증류 워크플로 (원본 → 증류 → 지침 반영)

1. 원본 ebook/PDF를 `sources/`에 넣는다.
2. 추출 → **실행가능 원칙만** 목록화 → 채널 금지규칙(`시사베테랑_대본_작성_시스템.md`)과 대조해
   **호환분만 채택** → `<주제>-원칙.md`로 저장.
   - ⚠️ "돈 되는 글" 류는 감정선동·CTA·낚시 경향 → 시사베테랑 금지(감정과잉·투자권유·신뢰도저하)와
     충돌. 충돌 원칙은 버린다.
3. 채택 원칙을 `siasa_writer.SYSTEM`(작성 규칙) / `OUTLINE`(구성)에 반영
   → 이후 **모든 대본 생성에 영구 적용**. (필요 시 원칙 구현 예시 대본 → corpus → LoRA 재학습)

> 증류는 자료 1건당 **1회 작업**이지만, 반영된 원칙은 매 생성에 상시 작동한다.

## 버전 고정 정책

정본 `.md`는 `프로그램지침&지식_16/` 원본의 **스냅샷**이다(편집은 원본에서 → 여기로 갱신). 정본을 단일
출처로 삼아 작가/디렉터가 동일 규칙을 참조하게 한다. (프로필 운영 사본과의 동기화는 drift 발견 시 처리.)
